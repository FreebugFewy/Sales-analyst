# Crypto Analyst Dashboard — Design Spec

**Date:** 2026-04-19
**Role:** Business Analyst, Operations & Strategy — Visa Growth Products & Partnerships
**Project type:** Fresh repo, Streamlit multipage app

---

## Overview

A 3-page Streamlit dashboard that models the operational analytics work performed by a Business Analyst embedded in Visa's Growth Products & Partnerships team. The portfolio covers Visa's real public crypto initiatives: USDC Settlement Rails, Visa x Coinbase Card, Visa x Crypto.com Card, and Crypto B2B Partnerships. The centrepiece is a Program Health Scorecard that ranks programs by weighted KPI score — the kind of tool a Chief of Staff opens every Monday to inform executive decisions.

The project directly demonstrates the role's core success criteria: clean and trusted data, exec-ready synthesis, and the ability to anticipate issues rather than react to them.

## Data Model

**File:** `streamlit_app/data/crypto_programs.csv`
**Rows:** ~480 (4 programs × 5 regions × 24 months)

| Column | Type | Description |
|---|---|---|
| `month` | date | Jan 2024 – Dec 2025 |
| `program` | string | USDC Settlement Rails, Visa x Coinbase Card, Visa x Crypto.com Card, Crypto B2B Partnerships |
| `region` | string | NA, EU, AP, LAC, MEA |

| `mom_growth` | float | Month-over-month volume growth rate |
| `partner_adoption_rate` | float | % of eligible partners actively transacting (0–1) |

## Pure Functions

### `compute_health_score(okr_attainment, mom_growth, partner_adoption_rate) → float`

**File:** `streamlit_app/utils/health_score.py`

Weighted composite score (0–100):
- OKR attainment: 40%
- Partner adoption rate: 35%
- MoM growth: 25%

Returns a float score and a band: `High` (≥70), `Medium` (40–69), `Low` (<40).

No Streamlit dependency. Fully unit-testable.

### `compute_risk_flags(program_df) → list[dict]`

**File:** `streamlit_app/utils/risk_flags.py`

Scans the most recent 3 months of data per program/region. Flags entries where:
- OKR attainment < 0.70 for 2+ consecutive months
- Partner adoption rate declining for 2+ consecutive months
- MoM growth negative for 2+ consecutive months

Returns a list of dicts: `{program, region, flag_type, recommendation}`. The `recommendation` field is a one-line exec-ready string (e.g. "Visa x Coinbase Card in MEA has missed volume OKR for 3 consecutive months — recommend partner enablement review.").

No Streamlit dependency. Fully unit-testable.

## Pages

### Page 1 — Portfolio Executive Summary (`1_portfolio_summary.py`)

**Purpose:** Monday-morning portfolio health view for a Chief of Staff.

- **KPI cards (top row):** Total portfolio volume (USD), blended OKR attainment %, portfolio MoM growth, total active partners
- **Volume trend chart:** Line chart of monthly volume by program (Altair, last 12 months)
- **Program Health Scorecard table:** Each program's latest health score, band (High/Medium/Low), OKR attainment, adoption rate, and MoM growth — sorted by score descending

### Page 2 — Program Deep-Dive (`pages/2_program_deep_dive.py`)

**Purpose:** Drill into a single program's performance and economics.

- **Program selector:** Dropdown (4 programs)
- **Volume trend:** Monthly volume with OKR target overlay (Altair)
- **OKR attainment over time:** Bar chart, coloured by attainment band
- **Partner adoption by region:** Horizontal bar chart (latest month)
- **Settlement Efficiency Model:** Cost-per-transaction curve vs. volume — shows the volume threshold at which the program becomes cost-efficient vs. traditional rails. Static model derived from synthetic cost assumptions, rendered as an Altair line chart with a break-even marker.

### Page 3 — Risk & Recommendations (`pages/3_risk_recommendations.py`)

**Purpose:** Surfaces issues before they become exec escalations.

- **Risk flags table:** All current flags from `compute_risk_flags()`, with program, region, flag type, and one-line recommendation
- **Flag summary KPI cards:** Count of High/Medium/Low risk flags across portfolio
- **Strategic Priorities section:** Static markdown block — 3 prioritised recommendations framed as a QBR talking point (e.g. MEA adoption gap, USDC rail volume ramp, B2B partnership pipeline). Written to read like an exec briefing note.

## Testing Strategy

Both pure functions are tested in isolation with no Streamlit dependency.

**`test_health_score.py`** (~6 tests):
- High/Medium/Low band boundaries
- Correct weighting (OKR-dominant vs adoption-dominant inputs)
- Edge cases: all zeros, all ones, partial inputs

**`test_risk_flags.py`** (~5 tests):
- Flags correctly when OKR attainment < 0.70 for 2+ months
- Flags correctly when adoption declining
- No false positives on healthy programs
- Recommendation string is non-empty for every flag
